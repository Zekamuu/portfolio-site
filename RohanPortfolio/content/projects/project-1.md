A weather station connected to IoT cloud, updateding cloud variables.



Description: This is my first project, where I used an ESP-32 Microcontroller to power a DHT-11 sensor and create a weather station connected to the cloud via Arduino's IoT cloud device service. I sought to introduce myself to the connection between hardware and software, and thought that this project would be a good start.

This was my first experience planning out the resrouces I would need, ordering parts, and assembling everything together. Though I imagined a simple project like this would be straightforward, many roadblocks showed up. One example is the pinout diagrams I was searching up online, as many of them were for different versions of the ESP-32 microcontroller. Therefor, I had to test using a multimeter to identify correct pinouts. 

The main takeaway from this project was how to troublshoot physical projects. Before, debugging purely software code involved a simple process of using an IDE software to run a debugger, and search line by line until the problem was found and resolved. The change here was that when a problem occured, I didn't know when it was a physical issue or software issue, which meant that I had to test an incredibly larger set of scenarios to fix problems, even though things were very simple. 
I gained experience in figuring out data flow through the chip and reading pinout diagrams, as well as writing a new type of software (code for hardware) and wiring.
